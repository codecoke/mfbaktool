1. 关于

FastCopy是 Windows 上最快的复制/备份软件。
它支持 UNICODE 和超过 MAX_PATH （260 个字符） 文件路径名。
它根据源和 DestDir 是否采用相同或不同的 HDD（或 SSD）自动选择不同方法。
差异硬盘读取和写入分别由单独的线程并行处理。
相同的硬盘读取被处理，直到大缓冲区填满。填充大缓冲区时，将批量开始和处理写入。
读取/写入处理时没有操作系统缓存，因此其他应用程序不会变慢。
它可以实现接近设备限制的读/写性能。
可以指定包括/排除筛选器（UNIX 通配符样式）。ver3.0 或更晚，可以指定相对路径。
它运行速度快，不会占用资源，因为未使用 MFC。（仅使用 Win32 API 和 C 运行时设计）
ver3.0 以后，FastCopy 的设置文件已自 FastCopy.ini 重命名到 FastCopy2.ini。

但是，一些 NAS 时间戳精度非常奇怪，因此提供手动设置（以 msec 表示）。

8. 警告

在某些模式下，源目录中具有相同文件名的目标文件将被强制覆盖，而不确认。
在同步模式下，在源目录中不存在具有相同文件名的目标文件将被强制删除，而不确认。
在删除模式下，目标文件和目录将被强制删除而不确认。
如果 FastCopy 有错误，则存在严重问题的可能性。
请自担风险使用。（见2.许可)
